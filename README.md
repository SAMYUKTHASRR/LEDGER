## Why

Final semester stacks assignments, series exams, attendance thresholds, and project/seminar milestones on top of each other with no single place to see all of it. Ledger exists to close that gap without needing an account, a server, or an internet connection.

## Features

- **Dashboard** — surfaces the single most urgent deadline and everything due in the next 7 days
- **Deadlines** — tracks assignments, submissions, exams, and lab records, each with a visual urgency indicator that intensifies as the due date approaches
- **Attendance** — logs classes held vs. attended per subject, flags anything under KTU's 75% minimum, and estimates classes needed to recover
- **Marks** — records series test and assignment scores per subject with a running average
- **Project & Seminar** — pre-built milestone chain (topic finalisation → seminar topic confirmation → guide approval → reviews → final submission) with editable notes per milestone

## Stack

Vanilla HTML, CSS, and JavaScript. No frameworks, no build step, no dependencies.

## Data & privacy

All data is stored in the browser's local storage on the device it's opened on. Nothing syncs, nothing is sent anywhere, and there's no login. Clearing browser data for this file will clear the tracker.

## Usage

1. Download `ledger.html`
2. Open it directly in any browser
3. Add subjects, deadlines, and milestones — data persists automatically between sessions

Works fully offline once the file is saved locally.

## Notes

- Attendance recovery estimate is approximate and assumes future classes are all attended
- Milestone list can be edited freely — the defaults are a starting structure, not a fixed requirement
